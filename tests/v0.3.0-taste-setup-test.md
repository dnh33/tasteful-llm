## Part 2: Post-Install Live Test (run in new session with plugin enabled)

### Instructions

After installing the plugin and restarting the chat session:

1. Run `/taste-setup`
2. Walk through each phase and verify the checklist below

### Phase Verification Checklist

#### Preamble
- [ ] Skill loads when `/taste-setup` is invoked
- [ ] No existing profile detected (first run)
- [ ] Scope question presented (Global vs. Project)
- [ ] Only ONE question asked per turn

#### Phase 0: Silent Scan
- [ ] Scans project README.md silently (no user prompt)
- [ ] Does NOT present precise statistics (no "average 11.3 words")
- [ ] Presents qualitative observations with quoted evidence

#### Phase 1: The Mirror
- [ ] Presents 4-5 specific observations about the project's prose
- [ ] Ends with one thing that CANNOT be determined from scanning
- [ ] Asks confirmation: "Does this feel like you?"
- [ ] Waits for response before continuing

#### Phase 2: Editorial Values
- [ ] Q1 asked: "When you edit other people's writing, what's the one thing you always fix?"
- [ ] Q2 asked: "If I wrote something technically flawless but still felt wrong..."
- [ ] Q3 asked OR skipped based on Phase 1 signal (adaptive)
- [ ] Q4 asked OR skipped based on Q2 specificity (adaptive)
- [ ] Each question on a separate turn
- [ ] No craft terminology used ("register", "cadence", "voice", "tonal")

#### Phase 2b: Creative Beliefs (opt-in)
- [ ] Opt-in prompt appears after Phase 2: "I can ask 3 questions about your creative philosophy..."
- [ ] If opted in: exactly 3 Rubin probes presented, one per turn
- [ ] Each probe shows the Rick Rubin quote + a question
- [ ] Response options: agree / disagree / complicated

#### Phase 3: The Aha Moment
- [ ] Two versions of a paragraph presented (A and B)
- [ ] BOTH versions are competent writing (B is NOT obviously bad slop)
- [ ] The difference between A and B is VOICE, not QUALITY
- [ ] Asks "Which sounds more like how you'd say it?"
- [ ] Topic is derived from the project's README (not generic)

#### Phase 4: The Characterization
- [ ] Prose characterization presented (a paragraph, NOT a bulleted list)
- [ ] Written in second person ("You write like someone who...")
- [ ] Specific and evidence-based (references actual answers/choices)
- [ ] Shows save path
- [ ] Offers adjustment opportunity

### Profile Output Verification

After /taste-setup completes, verify the generated profile:

```bash
# Check profile was created
cat ~/.tasteful-llm/taste-profile.md  # if global
# OR
cat .tasteful-llm/taste-profile.md    # if project

# Verify structure
grep "tasteful-llm-profile:v2" ~/.tasteful-llm/taste-profile.md
grep "## Voice DNA" ~/.tasteful-llm/taste-profile.md
grep "## Creative Beliefs" ~/.tasteful-llm/taste-profile.md
grep "## Learned Preferences" ~/.tasteful-llm/taste-profile.md
grep "## Confirmed Good" ~/.tasteful-llm/taste-profile.md
grep "## Anti-Preferences" ~/.tasteful-llm/taste-profile.md
```

- [ ] Profile file created at correct path (global or project, matching scope choice)
- [ ] Version marker: `<!-- tasteful-llm-profile:v2 -->`
- [ ] All 5 section headers present
- [ ] Voice DNA has 3-5 entries with `[setup:YYYY-MM-DD]` tags
- [ ] Creative Beliefs has entries (if opted in) with `[setup:YYYY-MM-DD]` tags
- [ ] Creative Beliefs is empty placeholder (if not opted in)
- [ ] Learned Preferences has placeholder comment only
- [ ] Confirmed Good has placeholder comment only
- [ ] Anti-Preferences has 2-4 entries with `[setup:YYYY-MM-DD]` tags and reasoning
- [ ] Total word count of setup entries is under 250 words (150 core + 100 beliefs)

### Quality Verification

These are subjective but critical — the whole point is taste:

- [ ] The characterization paragraph feels like RECOGNITION, not a horoscope
- [ ] The characterization is specific enough that it couldn't describe a random person
- [ ] The A/B aha moment felt like the system understood something real
- [ ] The questions felt like a creative director interview, not a settings panel
- [ ] The overall experience took 4-8 minutes
- [ ] You'd want to screenshot the characterization paragraph
